# 2026.3.0 Developer Blog Posts Plan

## Context

ESPHome 2026.3.0 includes several developer-facing breaking changes that need blog posts on developers.esphome.io. Blog posts are written for cross-cutting changes that affect external component developers — not for individual component changes or undocumented API changes.

GitHub code search confirmed that runtime_image/online_image (#10212) has zero external usage, so it was dropped.

## Blog Posts to Write (6 total)

### Blog Post 1: ESP8266 PROGMEM for Icon & Device Class Strings
**PRs:** #14437 + #14443
**Date:** 2026-03-12
**File:** `docs/blog/posts/2026-03-12-esp8266-progmem-icon-device-class.md`

**What changed:**
- `get_icon_ref()` and `get_icon()` deprecated on all platforms (removal 2026.9.0)
- `get_device_class_ref()` and `get_device_class()` deprecated on all platforms (removal 2026.9.0)
- On ESP8266: `static_assert` error — these methods cannot work because strings are in PROGMEM
- New buffer APIs: `get_icon_to(std::span<char, MAX_ICON_LENGTH>)` and `get_device_class_to(std::span<char, MAX_DEVICE_CLASS_LENGTH>)`
- Icon max length enforced at 63 characters

**Migration:**
```cpp
// Before
const auto &icon = entity->get_icon_ref();
const auto &dc = entity->get_device_class_ref();

// After
char icon_buf[MAX_ICON_LENGTH];
const char *icon = entity->get_icon_to(icon_buf);

char dc_buf[MAX_DEVICE_CLASS_LENGTH];
const char *dc = entity->get_device_class_to(dc_buf);
```

**Version compat:**
```cpp
#if ESPHOME_VERSION_CODE >= VERSION_CODE(2026, 3, 0)
  char icon_buf[MAX_ICON_LENGTH];
  const char *icon = entity->get_icon_to(icon_buf);
#else
  const auto &icon = entity->get_icon_ref();
#endif
```

### Blog Post 2: RP2040 Framework Update (Pico-SDK 2.0)
**PR:** #14328
**Date:** 2026-03-12
**File:** `docs/blog/posts/2026-03-12-rp2040-pico-sdk-2.md`

**What changed:**
- arduino-pico: 3.9.4 → 5.5.0
- pico-sdk: 1.5.1 → 2.1.0
- GCC 12 → GCC 14
- `padsbank0_hw` → `pads_bank0_hw` (pico-sdk 2.0 hardware register rename)
- `SerialPIO::NOPIN` → `NOPIN` (now global macro)
- `FS::info64` removed (use `FS::info`)
- Audio APIs return bytes instead of frames

**Who is affected:** External components targeting RP2040 that use old pico-sdk register names or arduino-pico APIs that changed.

**Backward compat shims exist** for most register renames. `SerialPIO::NOPIN` has no shim.

### Blog Post 3: Application name/friendly_name Now Return StringRef
**PR:** #14532
**Date:** 2026-03-12
**File:** `docs/blog/posts/2026-03-12-application-name-stringref.md`

**What changed:**
```cpp
// Before
const std::string &get_name() const;
const std::string &get_friendly_name() const;

// After
const StringRef &get_name() const;
const StringRef &get_friendly_name() const;
```

- `StringRef` has implicit `operator std::string()`, `.c_str()`, `.size()`, `.empty()`, so most code compiles unchanged
- Key action: `const std::string &name = App.get_name()` silently creates a heap-allocated temporary — update to `const auto &name = App.get_name()`
- Saves 752 bytes flash + 176 bytes RAM on ESP8266, 2,508 bytes flash on RP2040

**Migration:**
```cpp
// Before — compiles but silently heap-allocates:
const std::string &name = App.get_name();

// After — zero allocation:
const auto &name = App.get_name();
```

### Blog Post 4: http_request API Modernization
**PRs:** #14024 + #14027
**Date:** 2026-03-12
**File:** `docs/blog/posts/2026-03-12-http-request-api-modernization.md`

**What changed:**

PR #14024:
- Response headers: `std::map<std::string, std::list<std::string>>` → `std::vector<Header>`
- Collect headers: `std::set<std::string>` → `std::vector<std::string>` (must be pre-lowercased)
- `get_response_headers()` (returning full map) removed — use `get_response_header(name)` instead
- `start()` with `std::set` overload deprecated (removal 2027.1.0)
- `collect_headers` parameter renamed to `lower_case_collect_headers` to make contract explicit

PR #14027:
- Request headers: `std::list<Header>` → `std::vector<Header>` throughout `perform()`, `start()`, `get()`, `post()`
- Deprecated `std::list<Header>` overloads provided for `start()`, `get()`, `post()` (removal 2027.1.0)

**Migration:**
```cpp
// Headers — change one word
std::list<http_request::Header> headers;   // Before
std::vector<http_request::Header> headers; // After

// Response headers — use singular getter
auto headers_map = response->get_response_headers(); // Before (removed)
auto value = response->get_response_header("content-type"); // After
```

### Blog Post 5: Conditional Compilation of get_loop_priority()
**PR:** #14210
**Date:** 2026-03-12
**File:** `docs/blog/posts/2026-03-12-conditional-get-loop-priority.md`

**What changed:**
- `get_loop_priority()` virtual method now guarded behind `#ifdef USE_LOOP_PRIORITY`
- Only defined on RP2040 (the only platform where it has an effect)
- Was dead code on ESP32/ESP8266/LibreTiny since PR #9823
- Removes one vtable entry from every `Component` subclass on non-RP2040 platforms

**External usage:** ~11 external components override this method:
- Dilbert66/esphome-components, Marsupilami23/myesphometemplates, Protofy-xyz/esphome-components, SimoPk/deep_sleep_libretiny, brandond/esphome-tuya_pir, mansellrace/esphome-domotica-filare, motwok/thisandthat, sebirdman/m5paper_esphome, somebox/radio-retrofit-asc-tuner, standsi/esphome-external-components, szupi-ipuzs/esphome-external-lt-deepsleep

**Migration:**
```cpp
// Before
float get_loop_priority() const override { return 10.0f; }

// After
#ifdef USE_LOOP_PRIORITY
float get_loop_priority() const override { return 10.0f; }
#endif
```

**Version compat:**
```cpp
// Safe on all versions — #ifdef is a no-op when the define exists (pre-2026.3.0)
// and correctly guards the override when it doesn't (2026.3.0+)
#ifdef USE_LOOP_PRIORITY
float get_loop_priority() const override { return 10.0f; }
#endif
```

### Blog Post 6: Socket Abstraction Layer Devirtualized
**PR:** #14398
**Date:** 2026-03-12
**File:** `docs/blog/posts/2026-03-12-socket-devirtualization.md`

**What changed:**
- `socket::Socket` and `socket::ListenSocket` changed from virtual base classes to concrete type aliases
- Listen sockets now use `ListenSocket` type instead of `Socket`
- `Socket::ready()` converted to free function `socket_ready_fd()`
- Saves 1,020–3,228 bytes flash across platforms

**External usage:** 3 external components:
- Dilbert66/esphome-components, elupus/esphome-nibe, tomquist/marsrelay

**Migration:**
```cpp
// Listen socket type change
std::unique_ptr<socket::Socket> server_;                       // Before
std::unique_ptr<socket::ListenSocket> server_;                 // After

// Socket creation unchanged
server_ = socket::socket_ip_loop_monitored(SOCK_STREAM, 0);   // Same

// ready() method change
if (socket->ready()) { ... }          // Before
if (socket_ready_fd(fd)) { ... }      // After
```

**Version compat:**
```cpp
#if ESPHOME_VERSION_CODE >= VERSION_CODE(2026, 3, 0)
  std::unique_ptr<socket::ListenSocket> server_;
#else
  std::unique_ptr<socket::Socket> server_;
#endif
```
